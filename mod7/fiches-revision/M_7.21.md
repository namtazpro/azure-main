| Catégorie | A | B1 | B2 |
|-----------|:-:|:--:|:--:|

> Rappel des niveaux : **1** = familiarisation · **2** = connaissance générale + application · **3** = connaissance détaillée + analyse/correction.
> Le niveau **2** pour B1/B2 rend ce module important : savoir documenter et communiquer clairement conditionne la traçabilité et la sécurité.

## 🎯 À retenir en priorité
- Un rapport de maintenance doit permettre d’identifier qui a fait quoi, quand, selon quelle référence et avec quel résultat.
- La documentation doit être à jour, contrôlée, distribuée et identifiable par versions/révisions.
- ATA 100 structure les manuels par chapitres normalisés.
- Un rapport de dépannage doit contenir des faits observables, des données mesurables et une logique technique démontrable.
- La passation de consignes doit éviter toute interprétation : travaux en cours, systèmes ouverts, points sécurité, prochaine action.
- Communiquer ne signifie pas seulement informer : il faut vérifier la compréhension et sécuriser l’action.

## Synthèse du cours

## 1. Documentation

Le stagiaire doit être capable de rédiger un rapport de maintenance conforme et de documenter un dépannage de manière structurée.

### 1.1 Gestion de la documentation aéronautique

La documentation doit être mise à jour pour garantir sécurité et conformité réglementaire. Les mises à jour reflètent :
- modifications de conception ;
- améliorations de sécurité ;
- évolutions réglementaires ;
- révisions des manuels ;
- Service Bulletins.

Un système de contrôle documentaire garantit l’utilisation des versions récentes. Un système de repérage permet d’identifier les points modifiés ou créés, par exemple par **barre de révision**.

### 1.2 Structure ATA 100

L’**ATA 100** est un système de codification standardisé pour organiser la documentation technique.

Objectifs :
- uniformiser les manuels ;
- localiser rapidement l’information ;
- permettre une communication commune entre constructeurs, compagnies et maintenance.

### 1.4 Warning, Caution, Note

| Terme | Signification |
|---|---|
| **WARNING** | situation pouvant provoquer mort, blessures graves ou dommages matériels |
| **CAUTION** | situation potentiellement dangereuse pouvant entraîner lésions ou dommages |
| **NOTE** | explication complémentaire aidant à comprendre tâche ou système |

Ces mentions signalent limitations, restrictions de maintenance et avertissements sécurité.

### 1.6 Rapports de travaux

Objectifs :
- traçabilité complète ;
- conformité réglementaire ;
- sécurité des vols ;
- limiter erreurs facteurs humains.

Éléments obligatoires :
- identification : immatriculation, type appareil, date/heure ;
- références : Work Order, Task Card, AMM, IPC, SRM, numéro de révision ;
- travail effectué : description claire et factuelle ;
- pièces remplacées : P/N, S/N si applicable ;
- valeurs mesurées et tests ;
- certification : nom/trigramme, signature, licence/habilitation, date/heure.

Critères qualité :
- **exact** : données mesurables, références précises ;
- **complet** : aucune étape importante omise ;
- **lisible** : pas d’ambiguïté ;
- **conforme** : respect procédures organisme.

### 1.7 Rapports de dépannage

Objectif : tracer la logique de diagnostic et justifier l’action corrective.

Contenu :
- défaut constaté : message pilote, défaut technique, conditions ;
- diagnostic : tests, références TSM/FIM, résultats ;
- cause identifiée ;
- action corrective : remplacement/réparation + test satisfaisant.

Règle : documenter des faits, jamais des suppositions.

Un rapport doit préciser :
- zone ou système ;
- nature de l’anomalie ;
- conditions de découverte ;
- mesures relevées ;
- références documentaires ;
- inspections/contrôles/tests ;
- résultats obtenus ;
- éléments contrôlés conformes.

Analyse :
- cause identifiée et justifiée ;
- cause non confirmée ;
- analyse complémentaire requise.

Action corrective :
- nature de l’action ;
- référence de tâche ;
- résultat du test fonctionnel ;
- décision : remise en service, surveillance, analyse.

Principe fondamental : un rapport B1/B2 contient faits observables, données mesurables, logique technique démontrable. Il ne contient pas opinion personnelle, hypothèse non confirmée ou formulation approximative.

### 1.8 Passation de consignes

La **passation de consignes (Shift Handover)** doit permettre à l’équipe suivante de reprendre sans risque ni interprétation.

À transmettre :
- travaux en cours et avancement ;
- systèmes ouverts ou inopérants ;
- points critiques sécurité ;
- prochaine action.

Risques :
- fatigue liée aux shifts ;
- transmission incomplète ;
- supposition qu’une tâche est terminée alors qu’elle ne l’est pas.

Transmission efficace :
1. équipe sortante identifie et communique l’essentiel ;
2. équipe entrante comprend et assimile ;
3. processus formalisé avec lieu dédié.

Processus recommandé :
- réunion superviseurs sortants/entrants ;
- briefing collectif ;
- visites sur site ;
- état des tâches ;
- fiches utilisées ;
- problèmes et résolution ;
- événements inhabituels ;
- prochaines étapes.

## 2. Communication

Le stagiaire doit assurer une passation efficace et communiquer de manière claire, complète et concise.

### 2.1 Information vs communication

| Notion | Définition |
|---|---|
| **Information** | transmettre une donnée : tech log, défaut système, e-mail, consigne écrite |
| **Communication** | transmettre, faire comprendre et vérifier l’action attendue |

La communication inclut un **feedback** : questions, confirmation de compréhension, action identifiée.

Différence essentielle :
- informer = transmettre une donnée ;
- communiquer = transmettre, faire comprendre, sécuriser l’action.

### 2.2 Règle des 3 V

Règle d’Albert Mehrabian :
- **verbal** : mots, **7 %** ;
- **vocal** : ton, voix, **38 %** ;
- **visuel** : expression, posture, gestes, **55 %**.

En face-à-face, vocal + visuel représentent **93 %** de l’impact du message selon la règle présentée.

### 2.3 Principes clés

Une communication maintenance doit être :
- claire ;
- exacte ;
- complète ;
- vérifiée ;
- documentée.

Exemples :
- clair : « Inspection visuelle réalisée, aucune anomalie constatée. »
- exact : « Tâche effectuée selon AMM applicable. »
- exhaustif : « Panne identifiée, pièce en attente de remplacement. »
- vérifié : « Confirmez la poursuite de la tâche au prochain shift. »
- traçable : « Action enregistrée dans le tech log. »

Mauvaise communication :
- erreurs techniques ;
- mauvaise interprétation ;
- risque sécurité ;
- non-conformité réglementaire.

Facteurs humains :
- stress ;
- fatigue ;
- bruit ;
- barrière linguistique ;
- pression opérationnelle.

## ✅ Points clés à mémoriser
- Documentation à jour = sécurité + conformité + version contrôlée.
- ATA 100 organise les informations par chapitres normalisés.
- WARNING = risque mort/blessures graves/dommages ; CAUTION = danger potentiel ; NOTE = aide de compréhension.
- Rapport travaux : identification, références, travail, pièces P/N/S/N, mesures/tests, certification.
- Critères d’un bon rapport : exact, complet, lisible, conforme.
- En dépannage : faits observables, données mesurables, logique technique démontrable.
- Ne jamais écrire hypothèse non confirmée comme une cause.
- Shift handover : avancement, systèmes ouverts, points sécurité, prochaine action.
- Communication = message compris + action identifiée + feedback.
- Règle 3V : 7 % verbal, 38 % vocal, 55 % visuel.
- Mauvaise communication = risque sécurité et non-conformité.
- Attention aux faux amis : hazard = danger, assume = supposer, record = consigner, security = sûreté.
